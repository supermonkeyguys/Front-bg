

## HTTP 是什么

**HTTP 是一种用于在客户端和服务器之间传输超文本的应用层协议**


## HTTP 和 HTTPS 的区别

HTTP 是以明文形式发送内容，安全性低。而 **HTTPS 在 HTTP 基础上增加了 SSL / TLS 加密层，对数据进行加密和身份验证**，安全性更高


**加密算法**：

- **大数取模不可逆**
- “HTTPS 中的大数取模不可逆”是指：**在 RSA 等公钥加密算法中，对大整数进行模幂运算后，若没有私钥（或无法分解模数 n），几乎无法从密文反推出明文**。这种“计算上的不可逆性”是 HTTPS 安全通信的数学基石。



## HTTP 常见的请求头



| 字段名            | 说明                                   |
| -------------- | ------------------------------------ |
| Accept         | 能够接收的回应内容类型                          |
| Authorization  | 认证信息                                 |
| Cache-Control  | 指定指令来实现缓存机制                          |
| Content-Type   | 指示资源的多媒体类型                           |
| Content-Length | 请求体的长度（8位字节的倍数）                      |
| Cookie         | 服务器通过 Set-Cookie 发送的一个超文本传输协议 Cookie |
| Connection     | 浏览器想要优先使用的连接类型                       |
| Date           | 发送该消息的日期和时间                          |
| Expect         | 要求服务端要做的特定行为                         |
| Host           | 指定服务器域名和端口                           |
| If-Match       | 当客户端提供的实体与服务器上对应的实体相匹配时，才进行对应的操作。    |
| Origin         | 表示请求的来源                              |
| Range          | 只请求实体的一部分                            |
| User-Agent     | 浏览器身份标识字符串                           |


## HTTP常见状态码


**1XX**：请求已接收，继续处理

1. 100：服务器已接收请求头，客户端可继续发送请求体

2. 101：服务器根据客户端的请求切换协议，主要用于websocket

**2XX**：请求成功处理

1. 200(成功)：请求成功，返回对应数据

2. 201(已创建)：请求成功并且服务器创建了新的资源

3. 202(已创建)：服务器已经接受请求，但尚未处理

4. 204(无内容)：请求成功，但无返回内容


**3XX**：需要进一步操作完成请求

1. 301(永久移动)：资源永久迁移到新 URL（浏览器会缓存重定向）

2. 302(临时移动)：资源临时迁移到新URL

3. 305(使用代理)：请求者应该使用代理


**4XX**：请求存在错误

1. 400(错误请求)：请求参数错误或格式无效

2. 401(未授权)：请求需要身份验证

3. 403：服务器拒绝请求

4. 404：请求资源不存在

5. 405：请求方法不被允许


**5XX**：服务器处理请求时出错

1. 500：服务器内部未知错误

2. 502：网关/代理服务器收到无效响应

3. 503：服务器暂时不可用

4. 504：网关/代理服务器请求超时
